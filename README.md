# GridSense – Smart Power Grid Analytics & Fault Management

## Overview

GridSense is a polyglot persistence platform developed for the Advanced Data Management course.

The system integrates multiple database technologies to support real-time smart-grid monitoring, equipment management, consumer billing, network topology analysis and dashboard caching.

The implementation follows the architecture proposed in the accompanying report and demonstrates the use of different storage engines according to their strengths.

## Database Responsibilities

### PostgreSQL

Stores customer accounts and billing records requiring ACID guarantees.

### Cassandra

Stores high-volume time-series telemetry generated by smart meters and field sensors.

### MongoDB

Stores flexible equipment metadata with heterogeneous schemas.

### Neo4j

Represents the electrical grid topology and supports fault propagation analysis.

### Redis

Caches dashboard queries and active alerts.

## Running the Project

Clone the repository

```bash
git clone <repository-url>
cd gridsense
```

Start every service

```bash
docker compose up --build
```

The command automatically starts all database services together with the API and initialization routines.

---

## REST API

Example endpoints:

```http
GET /billing/{customer_id}
```

Returns customer billing information.

```http
GET /equipment/{equipment_id}
```

Returns equipment metadata.

```http
GET /faults
```

Returns active fault events.

```http
GET /telemetry/{sensor_id}
```

Returns telemetry measurements for a sensor.
